## Overview

This project implements and compares two initialization strategies for the k-means clustering algorithm from scratch in NumPy:

- **Random (Forgy)** — k centers drawn uniformly at random from the data
- **k-means++** — centers seeded sequentially using D² weighting, placing each new center proportionally to its squared distance from the nearest existing center

Both methods share the same Lloyd iterations, convergence criterion, evaluation metrics, stopping conditions, and random-seed schedule. The only variable is initialization, making the comparison controlled and attributable.

The analysis spans four experiments across three datasets, measuring inertia, ARI, NMI, variability, and convergence speed — and demonstrates empirically that k-means++ is **not unconditionally better**: its advantage is conditional on cluster structure and dimensionality.

## Key results

| Dataset | Mean J (random) | Mean J (k-means++) | ΔJ% | ARI (random) | ARI (k-means++) |
|---|---|---|---|---|---|
| Well-separated (2D, k=5) | 1109 | 439 | **−60.4%** | 0.833 | **0.966** |
| Overlapping (2D, k=5) | 2915 | 2931 | −0.5% | 0.780 | 0.768 |
| Digits (64D, k=10) | 70,493 | 71,381 | +1.3% | **0.512** | 0.464 |

- **Strong structure:** k-means++ outperforms random init on every axis — 60.4% lower inertia, ARI +0.13, half the Lloyd iterations, near-optimal rate 88% vs 42%.
- **Overlapping clusters:** Both methods tie on quality. k-means++ retains a stability benefit (CV 9.3% vs 14.5%) but no quality gain.
- **High dimensions:** The advantage reverses. Random init is marginally better on ARI and NMI; the result is consistent with distance concentration weakening the D² signal (plausible, not isolated here). Visible only through external metrics — inertia alone suggests equivalence.
- **Growing k:** The inertia ratio grows monotonically from 1.01 at k=2 to 1.82 at k=15, qualitatively consistent with the O(log k) bound (the ratio is a distinct quantity from the ln k factor and is not expected to match it numerically).

## Implementation details

All algorithms and metrics are implemented from scratch in NumPy. `scikit-learn` is used only to generate or load raw data (`make_blobs`, `load_digits`).

**k-means (Lloyd's algorithm)**
```python
def kmeans(X, k, init='random', random_state=None, max_iter=300, tol=1e-4):
    rng     = np.random.default_rng(random_state)   # one seeded generator per run
    centers = INITIALIZERS[init](X, k, rng)          # pluggable init strategy
    for i in range(1, max_iter + 1):
        labels      = assign_labels(X, centers)       # vectorised (n,k,d) argmin
        new_centers = update_centers(X, labels, k, rng)
        if np.max(np.linalg.norm(new_centers - centers, axis=1)) < tol:
            break
        centers = new_centers
    return centers, assign_labels(X, centers), compute_inertia(X, labels, centers), i
```

**k-means++ D² seeding**
```python
def init_kmeans_plus_plus(X, k, rng):
    centers = [X[rng.integers(0, len(X))].copy()]    # first center: random
    for _ in range(k - 1):
        D2   = dist_sq_to_nearest(X, np.array(centers))  # squared dist to nearest center
        prob = D2 / D2.sum()                              # normalise → probabilities
        centers.append(X[rng.choice(len(X), p=prob)].copy())
    return np.array(centers)
```

**Design decisions**

- **Pluggable initializer registry** — `INITIALIZERS = {'random': ..., 'kmeans++': ...}`. Both methods share every line of code except the single init call, making the comparison a true controlled experiment.
- **Seeded RNG threaded through all functions** — including empty-cluster recovery. Same `random_state` → byte-identical run every time.
- **Paired seeds** — both methods are evaluated on the same 50-seed sequence. Any observed difference is attributable to the initialization strategy, not lucky draws.
- **50 restarts** — k-means is non-convex; a single run can be lucky or unlucky. Fifty restarts expose the full distribution of outcomes and make stability claims defensible.
- **Standardize Digits only** — `make_blobs` clusters are already isotropic; Digits pixels have unequal intensity ranges that would otherwise dominate Euclidean distance.

## Evaluation metrics

| Metric | Type | What it measures |
|---|---|---|
| Inertia (φ) | Internal | Total squared distance of each point to its nearest center |
| CV (std/mean) | Stability | Run-to-run variability of inertia across 50 restarts |
| Near-optimal fraction (τ=1%) | Stability | Fraction of runs landing within 1% of the best inertia seen |
| ARI | External | Agreement with ground-truth labels, corrected for chance |
| NMI | External | Shared information between predicted and true labelings |

ARI and NMI are computed from scratch via a contingency table — no sklearn metric functions used.
